# ParkSight AI

**AI-Powered Parking Congestion Intelligence Platform**

*Detect → Quantify → Predict → Enforce*

## Overview
ParkSight AI is an advanced traffic operations platform designed to move law enforcement from reactive patrol to proactive, intelligence-driven operations. Rather than simply counting parking violations, ParkSight AI quantifies the **true traffic obstruction impact** of illegal parking, forecasts tomorrow's risk areas, and recommends targeted personnel deployment.

## Features
- **Data Quality Engine**: Handles violation deduplication, validation ratios, and calculates a Repeat Offender Index.
- **Spatial Intelligence**: Deploys HDBSCAN (with DBSCAN fallback) to identify dense, persistent violation hotspots.
- **Congestion Obstruction Proxy**: Evaluates traffic impact based on violation severity, peak-hour concentration, and junction proximity.
- **Growth Trend Score**: Tracks 7-day vs. 14-day growth to identify rapidly worsening zones.
- **Risk Forecast Engine**: Uses a 60% XGBoost + 40% LightGBM ensemble to predict tomorrow's violation risk, complete with expected % increase and a Forecast Confidence Score.
- **Enforcement Command Center**: Recommends the optimal Patrol Window (e.g., 08:00 - 11:00) and explicit resource allocation (e.g., 3 Traffic Officers, 2 Tow Vehicles) based on threshold logic.

## Repository Contents
- `ParkSight_AI_Pipeline.ipynb`: The core, industrial-grade data pipeline (data cleaning, HDBSCAN clustering, XGB+LGBM forecasting, risk scoring).
- `dashboard.py`: An interactive, multi-page Streamlit application acting as the executive Command Center.
- `outputs/`: Precomputed intelligence files generated by the pipeline for blazing-fast dashboard loads (< 2s).
- `requirements.txt`: Required Python packages.

## How to Run

### Prerequisites

* Python 3.10+ recommended
* Either:

  * **uv** (recommended) — fast Python package and environment manager
  * **pip** with a virtual environment

## Quick Start

```bash
uv venv
source .venv/bin/activate #.venv\Scripts\Activate.ps1 (for windows)
uv sync
streamlit run dashboard.py
```

### 1. Install Dependencies

#### Option A: Using uv (Recommended)

Create a virtual environment and install dependencies from `pyproject.toml`:

```bash
uv venv
```

Activate the environment:

**macOS / Linux**

```bash
source .venv/bin/activate
```

**Windows (PowerShell)**

```powershell
.venv\Scripts\Activate.ps1
```

Install the project:

```bash
uv sync
```

#### Option B: Using pip

Create and activate a virtual environment:

**macOS / Linux**

```bash
python -m venv .venv
source .venv/bin/activate
```

**Windows (PowerShell)**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

Install dependencies:

```bash
pip install -r requirements.txt
```

### 2. (Optional) Regenerate Pipeline Outputs

If you want to rerun the data pipeline:

1. Place `parking_violations.csv` in the project root.
2. Open and execute `ParkSight_AI_Pipeline.ipynb`.

This will regenerate the files inside the `outputs/` directory:

```text
outputs/
├── hotspots.csv
├── leaderboard_priority.csv
├── model_metrics.json
├── parking_hotspots_map.html
├── predictions.csv
└── station_ranking.csv
```

### 3. Launch the Dashboard

From the project root:

```bash
streamlit run dashboard.py
```

Once started, Streamlit will display a local URL (typically `http://localhost:8501`) in your terminal. Open it in your browser to access the dashboard.
